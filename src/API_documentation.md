# API Документация МКБ-сервиса

## Общая информация
Базовый URL: `easymed.pro`

API предоставляет два эндпоинта для работы с данными МКБ (Международной классификации болезней):
1. Поиск по МКБ
2. Получение детальной информации по коду МКБ

## Эндпоинты

### 1. Поиск по МКБ
Выполняет поиск по названию или коду МКБ.

**URL**: `/php/search.php`

**Метод**: GET

**Параметры запроса**:
- `q` (обязательный) - поисковый запрос (код или название болезни)
  - Должен быть закодирован с помощью `encodeURIComponent()`

**Пример запроса**:
```
GET /php/search.php?q=J20
```

**Успешный ответ**:
```json
[
    {
        "name": "Острый бронхит",
        "code": "J20"
    },
    // ... другие совпадения (до 100 результатов)
]
```

**Формат ответа**:
- Массив объектов, где каждый объект содержит:
  - `name`: Наименование болезни
  - `code`: Код МКБ
- Максимальное количество результатов: 100
- Кодировка: UTF-8

### 2. Получение детальной информации по коду МКБ
Возвращает подробную информацию о заболевании по его коду МКБ.

**URL**: `/php/get-data.php/login`

**Метод**: GET

**Параметры запроса**:
- `code` (обязательный) - код МКБ
- `username` (обязательный) - имя пользователя
- `password` (обязательный) - пароль пользователя

**Пример запроса**:
```
GET /php/get-data.php/login?code=J20&username=user&password=pass
```

**Успешный ответ**:
```json
{
    "child": {
        "code": "Код МКБ",
        "name": "Наименование болезни",
        "standards": [
            {
                "name": "Наименование стандарта или клиничекой рекомендации",
                "type": "Тип данных (одно из значений: 'Диагностика', 'Лечение')",
                "status": "Тип документа (одно из значений: 'Стандарт', 'Рекомендация')",
                "examinations": [
                    {
                        "id": "Внутренний идентификатор диагностической услуги",
                        "name": "Наименование диагностической услуги",
                        "comment": "Комментарий к услуге",
                        "examination_type_id": "Идентификатор типа обследования (1/2/3/4/5/6)",
                        "examination_stage_id": "Идентификатор этапа обследования (1/2)",
                        "is_required": "Признак обязательности (1/0)",
                        "is_qualitative": "Критерий качественности (1/0)"
                    }
                ],
                "treatments": [
                    {
                        "id": "Внутренний идентификатор лечебной услуги",
                        "name": "Наименование лечебной услуги",
                        "comment": "Комментарий",
                        "treatment_type_id": "Идентификатор типа лечения (1/2)",
                        "plan": "План лечения",
                        "duration": "Длительность",
                        "is_qualitative": "Критерий качественности (1/0)"
                    }
                ]
            }
        ]
    },
    "grownup": {
        // Аналогичная структура для взрослых
    }
}
```

Расшифровки численных значений:

"examination_type_id": 1 - Лабораторные исследования / 2 - Приемы специалистов / 3 - Инструментальные исследования / 4 - Лучевая диагностика / 5 - Ультразвуковые исследования / 6 - Хирургические манипуляции

"treatment_type_id": 1 - Лечебная манипуляция / 2 - Медикамент

"examination_stage_id": 1 - Первичный / 2 - Повторный

"is_qualitative": 1 - присутствует в критериях качества / 0 - не присутствует

"is_required": 1 - обязательное / 0 - по показаниям


**Ответ при ошибке аутентификации**:
```json
{
    "result": "denied"
}
```

**Формат ответа**:
- Объект с двумя основными разделами:
  - `child`: данные для детей до 18 лет
  - `grownup`: данные для взрослых старше 18 лет
- Каждый раздел содержит:
  - `code`: Код МКБ
  - `name`: Наименование болезни
  - `standards`: Массив данных по услугам
    - Каждый стандарт содержит:
      - `name`: Наименование стандарта или клиничекой рекомендации
      - `type`: Тип услуг
      - `status`: Статус документа
      - `examinations`: Массив обследований
      - `treatments`: Массив методов лечения

**Примечания**:
1. Все ответы возвращаются в формате JSON с кодировкой UTF-8
2. При отсутствии данных для определенной возрастной категории соответствующий раздел будет содержать пустой массив standards
3. Для успешного получения данных необходима корректная аутентификация
4. При неверных учетных данных возвращается ответ `{"result": "denied"}`
